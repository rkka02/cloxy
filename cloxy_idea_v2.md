# Cloxy: Local CLI Proxy

> Claude Code와 Codex CLI를 OpenAI 호환 로컬 API 서버처럼 감싸는 프록시 레이어

## 한 줄 정의

구독형 코딩 CLI를 `localhost` API 서버로 바꿔서, OpenAI 호환 툴이 로컬 CLI를 백엔드처럼 쓰게 만드는 프로젝트다.

## 문제 정의

대부분의 AI 코딩 툴은 OpenAI API 스펙을 전제로 만들어져 있다. 이 생태계에 Anthropic이나 OpenAI의 클라우드 API를 직접 연결하면 사용량이 늘수록 과금도 함께 늘어난다.

반면 Claude Code와 Codex CLI는 사용자가 이미 구독 중인 플랜 안에서 쓰는 경우가 많다. 이 프로젝트는 그 로컬 CLI를 OpenAI 호환 서버 뒤에 두어, 기존 툴이 별도 적응 없이 접속할 수 있게 만드는 것이 목표다.

## 제품 가설

- 로컬 CLI가 비대화형 호출과 세션 재개를 지원하면 프록시 구현은 가능하다.
- OpenAI 호환 툴은 완전한 API 호환보다 `chat completions` 호환만으로도 상당수 연결이 가능하다.
- 초기 버전은 "범용 에이전트 백엔드"보다 "호환 가능한 텍스트 백엔드"에 집중하는 편이 더 안정적이다.

## 현실적인 MVP 범위

1차 버전은 아래만 지원한다.

- `GET /v1/models`
- `POST /v1/chat/completions`
- SSE 스트리밍 응답
- Claude / Codex 선택형 백엔드

처음부터 하지 않는 것은 아래다.

- `/v1/responses`
- tool calling passthrough
- 이미지 입력
- embeddings
- 세션 유지 계층
- 완전한 Cursor 대체

## 왜 Claude를 먼저 타깃으로 하나

실사용 검증 기준에서 Claude Code는 다음이 더 명확했다.

- JSON 출력 모드가 명시적이다.
- 스트리밍 JSON 모드가 명시적이다.
- 세션 ID 재사용 플로우가 비교적 깔끔하다.

Codex CLI도 충분히 가능하지만, 현 시점에는 JSONL 이벤트를 직접 해석해야 하고 로컬 상태 경고가 더 잘 드러난다. 그래서 Cloxy의 첫 번째 안정화 백엔드는 Claude 쪽이 더 적합하다.

## 왜 세션을 안 넣는가

OpenAI 호환 `chat.completions`를 쓰는 대부분의 툴은 각 요청마다 대화 이력을 다시 보낸다. 즉, Cloxy가 별도로 세션을 유지하지 않아도 기본 동작은 성립한다.

오히려 프록시 레벨 세션은 다음 문제를 만든다.

- 세션 정리 시점이 애매하다.
- 클라이언트가 보낸 이력과 백엔드 CLI 내부 이력이 어긋날 수 있다.
- 디버깅이 어려워진다.

그래서 MVP는 stateless 서버로 간다. 각 요청은 독립적으로 처리하고, 필요한 문맥은 클라이언트가 보낸 `messages` 배열만 신뢰한다.

## 중요한 제약

문서 단계에서 가장 조심해야 할 오해는 이것이다.

> "CLI의 로컬 파일 접근과 MCP가 자연스럽게 OpenAI 호환 툴에 상속된다"

이건 자동으로 성립하지 않는다. 이미 많은 OpenAI 호환 코딩 툴은 자기 쪽에서 파일 컨텍스트와 에이전트 루프를 관리한다. Cloxy의 백엔드 CLI까지 독자적으로 파일을 읽고 수정하면 역할이 겹치고 결과가 불안정해질 수 있다.

그래서 MVP는 다음 전제를 둔다.

- 먼저 텍스트 백엔드로서 안정성을 확보한다.
- 백엔드 자체 툴 사용은 기본적으로 제한한다.
- 향후에만 옵트인 에이전트 모드를 추가한다.

## 비용 관점의 단서

이 프로젝트의 경제성은 "기술적으로 가능하냐"와 별개로 인증 방식에 달려 있다.

- 구독 기반 로그인 모드라면 고정비 프록시라는 가치가 생긴다.
- API 키 과금 모드라면 이득이 크게 줄거나 사라진다.

즉, Cloxy는 기술 프로젝트이면서 동시에 인증 모드에 따라 가치가 달라지는 제품이다.

## 개발 원칙

- 백엔드별 어댑터를 분리한다.
- OpenAI 호환 레이어는 최대한 얇게 유지한다.
- 서버는 stateless로 시작한다.
- 워킹 디렉터리는 명시적으로 제한한다.
- 비호환 동작은 "자동 보정"보다 "명시적 실패"를 택한다.
