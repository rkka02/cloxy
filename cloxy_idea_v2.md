# Cloxy : Local CLI Proxy — 아이디어 기획서

> Claude Code / Codex CLI를 OpenAI 호환 API 서버로 래핑해,
> 기존 오픈소스 코딩 툴을 구독료 고정으로 사용하는 프록시 레이어

---

## 문제

Cursor, Aider, Continue.dev 등 대부분의 AI 코딩 툴은 OpenAI API 스펙을 기반으로 동작한다.
Anthropic이나 OpenAI 모델을 이 툴들과 연결하려면 Cloud API를 직접 호출해야 하는데,
이 경우 구독료와 별개로 **토큰 사용량에 따라 비용이 계속 쌓인다.**

반면 Claude Code (Anthropic)와 Codex CLI (OpenAI)는 각사의 구독 플랜 안에서 동작하는 CLI 툴이다.
이미 구독료를 내고 있다면, 이 CLI를 통하는 것이 훨씬 경제적이다.

---

## 아이디어

**CLI 툴을 로컬 API 서버처럼 동작하게 만드는 중간 레이어를 두자.**

구체적으로는 내 컴퓨터 안에서 작은 서버를 하나 띄우고,
외부 툴이 `localhost`로 요청을 보내면 그걸 받아서 CLI에 전달하고,
CLI의 출력을 다시 OpenAI API 응답 포맷으로 변환해 돌려주는 것이다.

기존 툴 입장에서는 그냥 평범한 API 서버와 통신하는 것처럼 보인다.

---

## 적용 대상 CLI

### Claude Code (Anthropic)
- Anthropic Max 플랜 구독자 대상
- 로컬 파일시스템 접근, MCP 툴 연동 등 에이전트 기능이 강점
- 구독 플랜 한도 내에서 충분히 헤비하게 사용 가능

### Codex CLI (OpenAI)
- OpenAI ChatGPT Pro 또는 별도 Codex 구독 대상
- 터미널 기반 코딩 에이전트로 비슷한 포지션
- 동일한 프록시 레이어 구조로 연결 가능

두 CLI 모두 같은 방식으로 래핑할 수 있어서, 프록시 하나로 두 가지를 선택적으로 쓸 수 있는 구조도 가능하다.

---

## 어떤 툴들을 연결할 수 있나

OpenAI API 스펙을 따르는 툴이라면 이론상 전부 연결 가능하다.

- **Aider** — 터미널 기반 AI 코딩 어시스턴트
- **Continue.dev** — VS Code / JetBrains 익스텐션
- **Cursor** — API Base URL 변경 지원
- **Cline, Roo Code** — VS Code 에이전트 익스텐션
- 그 외 `--openai-api-base` 설정을 지원하는 오픈소스 전반

---

## 비용 관점

| 방식 | 비용 구조 |
|---|---|
| Cloud API 직접 연결 | 구독료 + 토큰 사용량 과금 |
| 이 프록시 방식 | 구독료 고정 |

사용량이 많을수록 이득폭이 커진다.
특히 자동화 파이프라인이나 장시간 에이전트 실행처럼 토큰을 많이 태우는 작업에서 유리하다.

---

## 이미 비슷한 게 있나

- **LiteLLM Proxy** — 다양한 백엔드를 OpenAI 호환으로 통합해주는 툴. 단, Cloud API 대상이라 토큰 과금은 그대로 발생한다.
- **Ollama** — 로컬 오픈소스 모델을 OpenAI 호환 서버로 래핑. 구조적으로 이 아이디어와 가장 유사한 선례.
- **claude-code-proxy 류 레포** — GitHub에 산발적으로 존재하지만 미성숙하고 관리가 안 되는 상태.

Codex CLI를 대상으로 한 프록시는 거의 전무한 상황.

---

## 핵심 가치

- 추가 비용 없이 기존 오픈소스 AI 코딩 생태계를 그대로 활용
- CLI가 가진 로컬 파일 접근, MCP 연동 등의 기능이 자연스럽게 상속됨
- 구독 플랜만 있으면 누구나 쓸 수 있는 구조
- Ollama가 로컬 모델에 해준 것을, 유료 CLI 툴에 적용하는 개념
